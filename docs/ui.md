# ease_fd — UI 명세

## 공통 레이아웃

### 셸 구조
모든 화면은 좌측 사이드바 + 우측 메인 영역으로 구성된다.

```
┌─────────────┬──────────────────────────────────────┐
│  사이드바    │  상단 topbar                          │
│  (200px)    │  프로젝트 헤더 + 탭 (프로젝트 화면만)  │
│             ├──────────────────────────────────────┤
│             │  콘텐츠 영역                           │
└─────────────┴──────────────────────────────────────┘
```

### 사이드바 구성
- 로고: `ease_fd` + 그룹명 (서브텍스트)
- 글로벌 네비: 대시보드, 프로젝트 검색
- 내 프로젝트 목록: 컬러 도트 + 프로젝트명 (최근 5개)
- 하단 푸터: 설정 링크 + 유저 아바타/이름/직책

### 프로젝트 헤더 탭 (프로젝트 내 모든 화면 공통)
```
홈 | 미팅 (N) | CPS | PRD | 설계 | 코드
```
- 완료된 단계: 기본 텍스트 색상
- 현재 단계: 파란색 + 하단 border
- 미완료 단계: 흐린 텍스트

### Topbar
- 좌측: breadcrumb (대시보드 / 프로젝트명 / 현재 화면)
- 우측: 화면별 주요 액션 버튼

## 1. 대시보드 `/dashboard`

### 구성
```
상단 지표 카드 (4개, 가로 배열)
└── 진행 중인 프로젝트 수 / 이번 주 미팅 수 / 미결 Pending 항목 수 / 오늘 CPS 자동 갱신 수

2열 레이아웃
├── 좌: 프로젝트 현황 패널
└── 우: 최근 미팅 패널 + Pending 항목 패널
```

### 프로젝트 현황 패널
- 각 프로젝트 행: 컬러 도트 + 프로젝트명 + 고객사명 + 단계 진행 바 + 상태 배지 + 최근 미팅 날짜
- 단계 진행 바: 미팅 → CPS → PRD → 설계 → 코드 → sync (6단계, 완료/진행/미완료 색상 구분)
- 클릭 시 프로젝트 상세로 이동

### 상태 배지 종류
| 상태 | 색상 |
|------|------|
| CPS 분석 중 | 주황 |
| PRD 작성 중 | 초록 |
| 설계 중 | 보라 |
| 코드 생성 중 | 파랑 |
| 완료 | 초록 |
| 아카이브 | 회색 |

### Topbar 버튼
- 미팅 기록 추가 (secondary)
- 새 프로젝트 (primary)

## 2. 프로젝트 상세 홈 `/projects/[id]`

### 구성
```
2열 레이아웃
├── 좌 (넓음)
│     ├── CPS 요약 패널 (Context / Problem / Solution 핵심만)
│     └── 최근 미팅 패널 (3개)
└── 우 (좁음)
      ├── 프로젝트 정보 패널 (고객사, 시작일, 미팅 수, CPS 버전, GitHub 연결 여부)
      ├── Pending 항목 패널 (3개 + 전체 보기)
      └── 파이프라인 진행 단계 패널 (단계별 완료/진행/미완료 아이콘)
```

### CPS 요약 패널
- Context / Problem / Solution 세 섹션
- 각 섹션 핵심 필드만 표시 (전체 내용은 CPS 탭에서)
- confidence 배지 표시 (suspected / probable / confirmed)
- 우측 상단 "전체 보기 →" 링크

### Topbar 버튼
- 미팅 추가 (secondary)
- GitHub Sync (secondary, 연결된 경우만 활성)

## 3. 미팅 목록 `/projects/[id]/meetings`

### 구성
```
마스터-디테일 레이아웃
├── 좌: 미팅 목록 (340px 고정)
└── 우: 미팅 상세
```

### 미팅 목록 (좌)
- 상단: 검색 입력창
- 각 항목: 제목 + 상태 배지 + 날짜 + 참석자 + 내용 미리보기 (2줄)
- 선택된 항목: 파란 배경 하이라이트
- 클릭 시 우측에 상세 내용 표시

### 미팅 상세 (우)
- 헤더: 제목 + 날짜/시간 + 참석자 + 분석 상태 배지
- 탭: 원본 내용 / CPS 업데이트 내역
- 원본 내용 탭: 마크다운 렌더링된 미팅 내용
- CPS 업데이트 내역 탭: 이 미팅으로 인해 변경된 CPS 필드 목록
- 분석 중일 때: 상단에 안내 배너 표시

### Topbar 버튼
- 미팅 추가 (primary)

## 4. 미팅 추가 `/projects/[id]/meetings/new`

### 구성
```
중앙 정렬 폼 카드 (max-width: 800px)
├── 폼 헤더: 제목 + 서브텍스트 ("저장하면 자동으로 CPS/PRD 분석 시작")
├── 폼 바디
│     ├── 미팅 날짜 (date picker) + 미팅 제목 (선택, 비우면 자동 생성) — 2열
│     ├── 참석자 (태그 입력, Enter로 추가, × 로 삭제)
│     └── 미팅 내용 (마크다운 에디터)
└── 폼 푸터: "저장 후 백그라운드에서 CPS 분석 시작" 안내 + 취소/저장 버튼
```

### 마크다운 에디터
- 입력 방식 탭: 직접 입력 / 파일 업로드 전환
- 직접 입력 탭:
  - 상단 탭: 편집 / 미리보기
  - 편집 탭 툴바: B / I / H2 / H3 / 목록 / 번호 / 인용
  - 편집 영역: 모노스페이스 폰트, min-height 260px
  - 미리보기 탭: 마크다운 렌더링 결과, 툴바 숨김
- 파일 업로드 탭: 드래그 앤 드롭 영역, .txt .md 지원
- 하단 힌트: "원본은 그대로 보존됩니다. 일반 텍스트로 입력해도 됩니다."

### Topbar 버튼
- 취소 (secondary)
- 저장 및 분석 시작 → (primary)

## 5. CPS 뷰어 `/projects/[id]/cps`

### 구성
```
3열 레이아웃
├── 좌: CPS 네비 (180px)
├── 중앙: CPS 섹션 카드들
└── 우: 버전 히스토리 패널 (220px)
```

### CPS 네비 (좌)
- 섹션 빠른 이동: Context / Problem / Solution / Assumptions / Risks / Out of Scope / Pending / Decision Log
- 버전 선택: v1.x.x 목록 (현재 버전 하이라이트)
- 클릭 시 해당 섹션으로 스크롤

### CPS 섹션 카드 (중앙)
각 섹션은 접힘/펼침 가능한 카드로 구성.

**섹션별 레이블 색상**
| 섹션 | 색상 |
|------|------|
| CONTEXT | 초록 |
| PROBLEM | 주황 |
| SOLUTION | 보라 |
| ASSUMPTIONS | 파랑 |
| RISKS | 빨강 |
| OUT OF SCOPE | 회색 |
| PENDING | 회색 |
| DECISION LOG | 회색 |

**각 섹션 카드 구성**
- 헤더: 섹션 레이블 + 섹션명 + 펼침/접힘 화살표
- 바디: 필드 목록 (필드명 + 값)
- hover 시 필드 우측에 "편집" 버튼 노출
- confidence 필드: 값 옆에 suspected / probable / confirmed 배지

**Pending 섹션**
- 체크박스로 완료 처리 가능
- 완료된 항목: 텍스트 취소선 + 흐린 색상
- 항목 타입 표시 (질문 / 인사이트 / 아이디어)

**Decision Log 섹션**
- 날짜 + 변경된 필드명 + 변경 이유 목록

### 버전 히스토리 패널 (우)
- 버전별 목록: 버전 번호 + 날짜 + 자동/수동 배지
- 현재 버전 하이라이트
- 하단: 메타 정보 (버전, 미팅 수, 마지막 갱신, 갱신 방식)

### Topbar 버튼
- 편집 (secondary)
- PRD 생성 → (primary, PRD 미생성 시) / PRD 보기 → (PRD 존재 시)

## 6. PRD 뷰어 `/projects/[id]/prd`

### 구성
```
3열 레이아웃
├── 좌: PRD 목차 네비 (180px)
├── 중앙: PRD 문서 본문 (max-width: 720px)
└── 우: 버전 히스토리 + 변경 내역 패널 (220px)
```

### PRD 목차 네비 (좌)
- 섹션 빠른 이동: 1. 개요 / 2. 목표 / 3. 사용자 / 4. 기능 요구사항 / 5. 비기능 요구사항 / 6. 범위 외 / 7. 미결 사항
- 버전 선택 목록

### PRD 문서 본문 (중앙)
- 상단: "CPS vX.X.X 기반으로 생성됨" 링크 배너 (CPS 뷰어로 이동)
- 문서 형태 레이아웃 (읽기 편한 prose 스타일)
- 기능 요구사항: FR 카드 형태
  - FR ID (FR-XXX-000)
  - 요구사항 제목
  - 상세 설명
  - 우선순위 배지: Must (빨강) / Should (주황) / Could (파랑)
- 미결 사항: 우선순위 도트 (빨강/주황/회색) + 제목 + 설명 카드

### 버전 히스토리 패널 (우)
- 버전별 목록: 버전 번호 + 날짜 + 기반 CPS 버전 + 자동/수동 배지
- 버전 간 변경 내역 (추가/수정된 FR 목록)
- 하단: 메타 정보 (버전, 기반 CPS, FR 수, 미결 사항 수, 마지막 갱신)

### Topbar 버튼
- 편집 (secondary)
- 설계 생성 → (primary)

## 공통 컴포넌트

### 배지 (Badge)
```
분석 중    주황 배경 / 주황 텍스트
완료       초록 배경 / 초록 텍스트
설계 중    보라 배경 / 보라 텍스트
코드 생성  파랑 배경 / 파랑 텍스트
아카이브   회색 배경 / 회색 텍스트

suspected  주황
probable   파랑
confirmed  초록

Must       빨강
Should     주황
Could      파랑

자동       초록
수동       회색
```

### 버튼
- primary: 파란 배경 (#185FA5), 흰 텍스트
- secondary: 흰 배경, 회색 border
- 위험 액션: 빨간 배경

### 패널 카드
- 흰 배경
- 0.5px border
- border-radius-lg
- 헤더 / 바디 / 푸터 구분선

### 빈 상태 (Empty State)
각 패널에 데이터 없을 때:
- 중앙 정렬 아이콘 + 설명 텍스트 + 액션 버튼
- 예: 미팅 없음 → "아직 미팅이 없어요. 첫 미팅을 추가해보세요." + 미팅 추가 버튼

### 로딩 상태
- 분석 중: 배지 + 상단 안내 배너
- 스켈레톤 로딩: 콘텐츠 로드 전 회색 placeholder

## 7. 새 프로젝트 `/projects/new`

### 구성
```
중앙 정렬 폼 카드 (max-width: 640px)
├── 폼 헤더: 제목 + 서브텍스트
├── 폼 바디
│     ├── 프로젝트명 + 고객사명 — 2열
│     ├── 프로젝트 색상 (컬러 도트 7개 선택)
│     ├── 설명 (textarea, 선택)
│     ├── 태그 (태그 입력, Enter로 추가)
│     ├── 구분선 "선택 설정"
│     ├── GitHub 레포 연결 (레포명 입력 + 자동 생성 버튼)
│     └── 생성 즉시 첫 미팅 추가 (토글)
└── 폼 푸터: 취소 / 프로젝트 생성 버튼
```

### 프로젝트 색상
- 7가지 컬러 도트 선택 (초록, 파랑, 보라, 주황, 코랄, 핑크, 회색)
- 선택된 도트: 외곽선 강조
- 사이드바 도트 색상으로 사용됨

### GitHub 레포 연결
- 레포명 입력 필드 + "자동 생성" 버튼
- 비워두면 프로젝트명 기반으로 자동 생성
- GitHub 미연결 시: 힌트 텍스트로 설정 → GitHub 링크 안내

### 생성 즉시 첫 미팅 추가 토글
- ON: 프로젝트 생성 후 바로 미팅 추가 화면으로 이동
- OFF: 프로젝트 상세 홈으로 이동

### Topbar 버튼
- 취소 (secondary)
- 프로젝트 생성 (primary)

## 8. 설정 `/settings`

### 구성
```
2열 레이아웃
├── 좌: 설정 네비 (180px)
└── 우: 설정 콘텐츠 영역
```

### 설정 네비 (좌)
섹션 클릭 시 우측 콘텐츠 전환 (SPA 방식).
```
계정
  └── 프로필
  └── 팀 · 그룹

연동
  └── GitHub

자동화
  └── CPS/PRD 분석 방법
  └── 미팅 저장 후 자동 분석 요청

환경설정
  └── 언어 · 표시
```

### 프로필 섹션
- 아바타 + 이름 + 이메일 + "사진 변경" 버튼
- 이름 / 직책 편집 (2열)
- 이메일: 읽기 전용 (변경 불가 안내)
- 비밀번호 변경 카드: 현재 비밀번호 + 새 비밀번호 + 확인 (2열)

### 팀 · 그룹 섹션
- 그룹명 + 그룹 ID + 내 역할 배지 (관리자/멤버)
- 팀원 목록: 이름 + 이메일 + 역할 + 내보내기 버튼
- 팀원 초대: 이메일 입력 + 초대 버튼
- 초대 코드 카드: 코드 표시 (모노스페이스) + 복사 + 재생성 버튼

### GitHub 섹션
- 연결된 경우: 초록 배경 카드 + GitHub 아이콘 + 계정명 + 연결 해제 버튼
- 미연결인 경우: GitHub OAuth 연결 버튼
- 기본 설정 카드:
  - 기본 레포 가시성 (Private/Public 셀렉트)
  - 레포명 접두사 (텍스트 입력, 선택)

### CPS/PRD 분석 방법 섹션
- SMART 분석 : 새 미팅 + 기존 CPS (5 credit 사용)
- FULL 분석 : 모든 미팅의 개별 요약본 + 새 미팅 (8 credit 사용)

### 미팅 저장 후 자동 분석 요청 섹션
- 미팅 저장 후 자동 분석 요청 토글 
- 체크를 하게 되면 CPS 분석 , PRD 분석 을 선택할 수 있다(체크박스)

### 언어 · 표시 섹션
- 언어 카드: 한국어 / English 선택 카드 (2열 그리드, 선택된 것 파란 테두리)
- 시간대 · 날짜 형식 카드:

## 9. PRD 뷰어 — 칸반 `/projects/[id]/prd`

### 구성
```
사이드 네비 (160px) + 메인 콘텐츠 + 우측 상세 패널 (280px)
```

### 상단 툴바
- 좌측: 문서 / 칸반 뷰 전환 탭 + CPS 버전 링크
- 우측: 총 FR 수 + FR 추가 버튼

### 칸반 보드
- Must / Should / Could 3열 고정
- 각 열 헤더: 컬러 배경 + 이름 + 카드 수 배지
  - Must: 빨강 (#FCEBEB / #791F1F)
  - Should: 주황 (#FAEEDA / #633806)
  - Could: 파랑 (#E6F1FB / #0C447C)
- 열 바디: 연한 배경, 카드 세로 스택

### FR 카드
- FR ID (모노스페이스, 흐린 색)
- 제목 (font-weight 500)
- 설명 2줄 말줄임
- 태그 (작은 pill)
- 클릭 시 우측 상세 패널 활성화 + 카드 파란 테두리

### 우측 상세 패널
카드 클릭 시 표시. 기본은 "카드를 선택하세요" 빈 상태.
- 헤더: FR ID + 제목 + 우선순위 배지
- 섹션: 설명 / 태그 / 관련 리스크 / CPS 연결 (해당 CPS 필드 링크)
- 관련 리스크 없으면 섹션 숨김

### 사이드 네비 (좌)
- 섹션: 개요 / 목표 / 사용자 / 기능 요구사항 / 비기능 / 미결 사항
- 기능 요구사항 선택 시 칸반 표시
- 나머지 섹션은 문서 뷰로 전환
- 하단: 버전 목록

### Topbar 버튼
- 편집 (secondary)
- 설계 생성 → (primary)

## 10. 설계 뷰어 `/projects/[id]/design`

### 구성
```
프로젝트 헤더 + 설계 탭 바 + 탭별 콘텐츠 영역
```

### 설계 탭 바
프로젝트 헤더 바로 아래 수평 탭. 탭 전환 시 콘텐츠 영역만 교체.
```
시스템 아키텍처 | 데이터 모델 | API 명세 | 프론트엔드 | 백엔드 | 보안
```

---

### 탭 1. 시스템 아키텍처

**데이터 흐름 다이어그램**
- 컬러 pill 노드 + 화살표로 시스템 흐름 시각화
- 배경: 연한 회색 카드 안에 배치

**2열 그리드 카드**
- 기술 스택: 영역 ↔ 기술 key-value 목록
- 배포 구조: 영역 ↔ 값 key-value 목록

**주요 설계 결정 카드**
- 결정 제목 (파란색) + 이유 설명 목록
- 결정 항목 간 구분선

---

### 탭 2. 데이터 모델

**ERD 테이블 뷰**
- 가로 스크롤 가능한 컨테이너
- 각 테이블: 헤더(파란 배경) + 필드 목록
- 필드: 이름(모노스페이스) + PK/FK 배지 + 타입
- 테이블 간 연결선(—) 표시
- 하단: 핵심 설계 포인트 설명 카드

---

### 탭 3. API 명세

**도메인 필터 버튼** (전체 / 재고 / 센터 / 동기화 등)
- 선택된 도메인에 해당하는 행만 표시

**API 테이블**
- 열: 메서드 배지 / 엔드포인트(모노스페이스) / 설명 / 도메인
- 메서드 배지 색상: GET 초록 / POST 파랑 / PATCH 주황 / DELETE 빨강
- 행 hover 시 배경 변화
- 도메인 필터로 즉시 필터링

---

### 탭 4. 프론트엔드

**2열 레이아웃**
- 좌: 폴더 트리 (모노스페이스, 디렉토리 강조)
- 우: 상태 관리 카드 + 화면 흐름 카드

---

### 탭 5. 백엔드

**2열 레이아웃**
- 좌: 폴더 트리 (routers / services / collectors / jobs)
- 우: 스케줄 동기화 카드 + 오류 처리 카드

---

### 탭 6. 보안

**2열 그리드**
- 인증/인가 카드: 역할별 접근 권한 key-value
- 데이터 보호 카드: 보안 정책 key-value

**Firestore 보안 규칙 카드**
- 모노스페이스 텍스트로 규칙 원칙 표시

### Topbar 버튼
- 편집 (secondary)
- 코드 생성 → (primary)

```
  └── 파이프라인
 
환경설정
  └── 언어 · 표시
```
 
### 자동화 > 파이프라인 섹션
 
**카드 1. 미팅 분석 기본값**
- 설명: 미팅 추가 화면에서 변경 가능. 여기서는 기본값만 설정.
- 컴포넌트: 라디오 그룹 (2개 옵션)
  - 스마트 분석 (기본값) — 5코인
  - 전체 재분석 — 8코인
- 선택된 항목: 파란 테두리 + 파란 배경
**카드 2. 자동 갱신**
- 설명: CPS 업데이트 시 하위 문서 자동 재생성 여부
- 컴포넌트: 토글 2개
  - PRD 자동 재생성 (기본: OFF)
    - OFF 시: 알림만 표시, 수동 실행
  - 설계 문서 자동 재생성 (기본: OFF)
    - OFF 시: 알림만 표시, 수동 실행
- 하단 info 박스: "자동 재생성 시 코인이 자동 차감됩니다. 잔액 부족 시 알림 후 중단."
**카드 3. 코인 사용 알림**
- 컴포넌트: 토글 2개
  - 분석 실행 전 확인 팝업 (기본: ON)
  - 잔액 부족 경고 — 50코인 이하 시 (기본: ON)
- 하단: 저장 버튼
